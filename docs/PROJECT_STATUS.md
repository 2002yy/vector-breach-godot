# Vector Breach Project Status

Updated: 2026-07-23

## 当前主线

- Godot 4.6 是当前可玩主线；旧 Babylon/WebGL 实现仅作历史参考。
- 当前目标不是逐帧复制旧引擎，而是保留经典 CS 的跳跃、蹲跳、枪械纪律和爆破决策，并采用现代战术射击游戏更清晰的地面启停与受击减速。
- 所有地图共享同一套玩家移动、跳跃、雷达范围和战斗规则。

## 已完成

- 移动：默认奔跑、Shift 静步、蹲伏慢行；普通跳约 1.25–1.32 m、蹲跳约 1.55 m；无二段跳、无自动翻越。
- 手感：高地面加速度、快速松键减速、反向急停、有限空中修正、贴边滑动、落地精度惩罚、受击减速和坠落伤害。
- 通行测试：0.4 / 0.65 / 0.8 / 1.3 / 1.55 m 分级验证，以及 20 cm 实际台阶通行。
- 战斗：完整 30 发步枪喷射、手枪半自动、距离衰减、头/胸/腹/手臂/腿命中部位、独立防弹衣/头盔、玩家死亡。
- 穿透：按玻璃、木材、石膏板、混凝土、金属区分阻力，并计算入口、出口和实际厚度；厚墙会阻断穿透。
- 回合：T/CT 可选阵营、冻结购买、持续安装/拆除、拆弹钳时长差异、可掉落/拾取世界 C4、变频蜂鸣、遮挡衰减爆炸伤害和自动下一回合。
- 经济：16000 金钱上限、连败补偿、安包失败补偿、目标行为奖励、武器差异击杀奖励、购买区/冻结期限制、持有物品上限与生存保枪。
- 装备：刀具和持刀速度、枪械掉落/拾取、HE/闪光/烟雾的购买、投掷、物理反弹、爆炸/视角闪光/烟雾遮挡。
- 雷达：圆形局部雷达，默认 24 m；支持旋转开关和 16–40 m 范围设置；显示地图边界、目标区、队友接口、合法侦测敌人、高低差、死亡标记和 C4 状态。
- HUD：比分、时间、存活人数、生命、护甲、金钱、弹药、Tab 计分板、击杀信息、C4 交互进度和闪光反馈。
- 设置：中文鼠标灵敏度、主音量、准星间距/长度、动态准星、雷达范围和雷达旋转。
- 环境语义：地图 JSON 可定义梯子、浅水和深水体积；运行时生成 `Ladder Area3D` / `Water Area3D`，支持上下攀爬、横向限速、梯顶离梯、跳离、涉水减速、深水浮力、水下状态和独立涉水脚步。
- 战术单位：新增可碰撞的灰盒 T/CT `CharacterBody3D` 胶囊，具备生命、护甲、头盔、武器字段、五类命中部位、死亡、出生、队伍存活计数和雷达信息；友方单位会真实阻挡玩家。
- AI 敌人：地图战斗目标默认启用可配置 Bot；所有地图路线会合并成双向连接图，Bot 可按 A* 路径巡逻和绕开直线阻挡，并具备视野距离/FOV/遮挡/烟雾判断、脚步/落地/枪声听觉记忆、反应与瞄准延迟、急停、短点射、弹匣和换弹，复用玩家伤害、空间枪声、材质脚步、梯子和水体规则。
- 云端协作：状态文档已进入仓库 `docs/PROJECT_STATUS.md`；新增根目录 `AGENTS.md`、Copilot 仓库指令、云端 Godot 4.6.2 安装步骤、Linux 测试脚本与 GitHub Actions 原生测试工作流。
- Foundry 视觉：保留 PBR 墙地材质和原碰撞拓扑，新增 9 条地面伸缩缝、7 片路线磨损区和 6 组贴墙检修覆板，用分区与结构接缝减弱大面积墙地板的灰盒感。
- Depot 视觉：不改地图 JSON 与碰撞拓扑，在原 PBR 资产上增加 38 段落地墙基、长墙分仓肋与顶帽、8 组贴墙检修覆板、9 条地面伸缩缝、7 片路线磨损区和 3 条排水格栅；北长廊、仓库、中路、服务道与目标区保留各自的工业材质识别。

## 敌人/队友实体当前边界

### 已具备

- 地图 JSON 可定义单位名称、T/CT 阵营、出生位置、朝向、护甲、头盔和武器字段；`friendly` / `enemy` 会根据本地玩家阵营自动解析。
- 使用 `CharacterBody3D` 胶囊参与地图、玩家和队友碰撞；当前友方单位会真实阻挡玩家。
- 共用现有伤害模型，支持头、胸、腹、手臂、腿五类命中部位，以及护甲、头盔、爆头和穿透后的伤害结算。
- 支持受伤、死亡、关闭碰撞、移除实体、队伍存活人数变化、敌方击杀奖励和淘汰胜负判断；友军伤亡不会发放击杀金钱。
- 友方单位始终进入雷达；敌方单位只有在距离、视角、无遮挡和无烟雾阻断等合法侦测条件满足后才短暂显示。
- 当前使用蓝色 CT、橙色 T 的胶囊和简单武器灰盒，仅用于验证碰撞、伤害、队伍和雷达协议。
- `combatTargets` 默认运行 AI；可通过 `aiEnabled: false` 保留静态训练靶。`teamActors` 当前默认不启用 AI，避免把雷达/阻挡协议单位误当成完整队友。
- Bot 会把 `aiRoutes` 与现有 `routes` 合并为共享连接图；重合节点自动合并，路线相邻点建立双向边，追查声音与巡逻时使用 A* 路径并定期重算。Foundry 当前实测生成 46 个导航节点。
- Bot 可合法发现本地玩家，烟雾会阻断视线；玩家普通/静步脚步、落地和枪声会按不同半径写入听觉记忆。
- Bot 发现目标后经历反应/瞄准阶段，急停至射击速度门限，再按短点射和间隔开火；弹匣耗尽后会换弹。命中共用玩家命中部位、护甲、受击减速与死亡链。
- AI 枪声和脚步已接入现有空间音频，AI 击杀可写入击杀信息。

### 尚未具备

- 当前是连接路线图 + 周期重规划 + 局部侧移，不是完整导航网格；没有掩体代价、危险区、跳跃连接、离路线自由寻路和复杂高低差择路，遇到未标注死角仍可能卡住。
- 感知尚未处理受伤来源、投掷物声响、队友共享信息、报点或 C4 状态。
- 会急停瞄准、短点射和换弹，但不会学习完整压枪、切枪、蹲伏、躲闪、架枪、撤退或按武器距离改变战法。
- 不会购买、保存或拾取武器，不会投掷 HE/闪光/烟雾，也不会携带、掉落、拾取、安装或拆除 C4。
- 没有队伍战术、报点、协同推进、补枪、回防、保枪和经济局策略。
- 没有正式角色模型、骨骼，以及持枪、移动、开火、换弹、受击和死亡动画；敌人已有空间脚步/枪声事件，但仍使用合成音，且没有语音表现。
- 玩家死亡后仍按单机训练逻辑结束本地回合，不能观察或接管仍存活的 AI 队友。
- 没有多人服务器权威、状态复制、延迟补偿、命中校验和断线重连。
- 计分板尚未维护每个战术单位的击杀、助攻、死亡、金钱、装备和网络状态。

- AI Bot 测试覆盖路线配置、冻结期限制、巡逻、听声调查、连接图绕墙、视觉发现、急停点射、玩家伤害、梯子和深水通行。

## 仍未完成

- 目前仍是单机训练/规则验证原型：已有最低可交火 AI 敌人，但没有完整导航网格、爆破决策、队伍协同、正式角色模型、联网权威同步、真人队友/敌人和整队经济库存。
- Foundry 与 Depot 已完成本轮墙地面结构层次提升；Gatehouse、Core Vault 和碰撞校准图仍主要使用程序化盒体和纯色材质，后续应按地图优先级逐张建立独立视觉层，而不是改动共享碰撞。
- 梯子/水体运行时规则已经全地图共用，但目前只在碰撞校准图放置了实际语义体积；其他正式地图需要在关卡设计确认后按 JSON 逐处标注，避免与现有视觉几何错位。
- 烟雾目前具备投掷、持续体积、视觉遮挡、雷达侦测阻断和 AI 视线阻断；后续仍需更高质量粒子体积、边缘可见性与穿烟记忆调优。
- 战术工具和经济已形成最小闭环，但还需要实际多人地图上的长局平衡与声音传播复测。
